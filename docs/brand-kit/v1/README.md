# BRAND KIT — ARIAS GROUP CARIBE · v1

**Versión:** v1 (2026-04-18)
**Para:** Equipo de marketing, diseño y agencias externas

---

## Qué es este kit

Todo lo que necesitas para diseñar cualquier pieza de marca Arias Group Caribe (redes sociales, banners, presentaciones, firmas de email, brochures, rotulación, eventos) manteniendo coherencia visual y tonal con la web y los documentos corporativos.

**Regla de oro:** si algo que vas a diseñar contradice este kit, prevalece el kit. Ante un caso no previsto, consultar antes de ejecutar.

---

## Estructura del kit

### 📘 01_guias/
- `brief_marca_arias_group_v1.md` — Posicionamiento, audiencia, tono de voz, competencia, principios de decisión. **Empezar por aquí.**
- `componentes_visuales.md` — Patrones visuales de la web (heros, grids, pull quotes, números, botones, CTAs). Aplicables a cualquier pieza.

### 🎨 02_logos/
- `arias_group_logo.svg` — Logo principal vectorial
- `arias_group_logo_500px.png` · `1000px.png` · `2000px.png` — Versiones raster
- `fassa_bortolo_logo.svg` + versiones raster — Para co-branding cuando aplique

**Nota:** los azules internos del logo (#1E4A8C, #7BA7C7) **viven solo dentro del logo**. No se usan fuera.

### ✍️ 03_tipografia/
- `tipografia.md` — Especificación de Fraunces (display) + Inter (UI), pesos, tamaños, cuándo usar cada una, qué no hacer.

### 🟦 04_paleta/
- `paleta_arias_group_caribe.html` — Carta de colores visual, interactiva (abrir en navegador)
- `tokens.json` — Paleta en formato de tokens para importar a Figma, Webflow, código, etc.

### 📝 05_copy/
- `copy_home_v1.md` — Todo el texto de la home de ariasgroupcaribe.com con contexto de sección. Úsalo como referencia de tono de voz al escribir copy para cualquier pieza nueva.

### 🌐 06_sitio_web_ejemplo/
- Copia compilada del sitio web actual. Doble-clic en `index.html` para ver en navegador. Útil para ver el sistema de diseño aplicado en contexto real.

## Flujo recomendado para diseñar una pieza nueva

1. **Leer el brief** (`01_guias/brief_marca_arias_group_v1.md`) — especialmente los 5 principios de decisión.
2. **Consultar los componentes** (`01_guias/componentes_visuales.md`) — ver si hay un patrón que ya encaje.
3. **Tomar colores de `tokens.json`** — copiar hex, nunca inventar.
4. **Usar Fraunces + Inter** con los pesos indicados — descargar ambas de Google Fonts.
5. **Escribir copy con el tono del brief** — sobrio, técnico, directo. Dato > adjetivo.
6. **Validar** contra los 5 principios antes de entregar.

---

## Los 5 principios de decisión (síntesis del brief)

1. **Seriedad > simpatía**
2. **Dato > adjetivo**
3. **Menos > más**
4. **Real > prometido** (nunca prometer lo que no se ejecuta ya)
5. **Cuando hay duda, preguntar**
